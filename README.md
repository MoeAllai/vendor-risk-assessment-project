# Vendor Risk Assessment Project

## Project Overview

This project simulates a Junior GRC Analyst assignment focused on third-party risk management for a mock SaaS company evaluating a new payroll vendor.

The goal of this project is to demonstrate practical GRC skills in vendor due diligence, security questionnaire review, SOC 2 report review, risk rating, findings documentation, remediation planning, and executive reporting.

## Mock Scenario

CloudCore Analytics is a 120-employee SaaS company planning to onboard a new third-party payroll vendor called PayLink HR Solutions.

Because the vendor will process sensitive employee data, including names, addresses, bank account details, salary information, and tax records, the vendor must complete a security and compliance review before onboarding.

## Project Scope

The assessment covers the following third-party risk areas:

- Vendor onboarding
- Data sensitivity review
- Security questionnaire review
- SOC 2 report review
- Access control review
- Encryption review
- Incident response review
- Business continuity review
- Privacy and data handling review
- Contract and security requirements
- Risk rating
- Remediation tracking

## Key Findings Summary

The review identified several vendor risk concerns, including:

- SOC 2 report has exceptions related to access review evidence
- Vendor does not provide customer-managed encryption keys
- Incident notification SLA is not clearly defined in the contract
- Vendor uses subcontractors but does not provide complete subcontractor review evidence
- Business continuity test results are older than 12 months
- MFA is enforced for employees but exception tracking needs improvement
- Data retention and deletion process needs clearer documentation

## Analyst Conclusion

PayLink HR Solutions may be approved with conditions because the vendor has a mature security program overall, but several risks should be remediated or contractually addressed before full onboarding.

The highest-priority items are SOC 2 exceptions, incident notification requirements, subcontractor oversight, data retention, and business continuity evidence.

This project demonstrates how a Junior GRC Analyst can support vendor risk management by collecting vendor evidence, reviewing questionnaire responses, identifying risks, documenting findings, and recommending approval conditions.
